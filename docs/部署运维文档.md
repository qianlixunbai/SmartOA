## 2. 快速启动（开发环境）

### 2.1 数据库初始化

```bash
# 1. 启动 MySQL 服务
# 2. 创建数据库
mysql -u root -p -e "CREATE DATABASE IF NOT EXISTS smartoa DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

# 3. 依次执行迁移脚本
mysql -u root -p smartoa < docs/mysql-p0-upgrade.sql
mysql -u root -p smartoa < docs/mysql-p1-upgrade.sql
mysql -u root -p smartoa < docs/mysql-p3-bcrypt.sql
mysql -u root -p smartoa < docs/mysql-p4-parallel.sql
mysql -u root -p smartoa < docs/mysql-p5-timeout.sql
```

迁移脚本说明：

| 脚本 | 内容 |
|---|---|
| `mysql-p0-upgrade.sql` | 建库 + 4 张基础表 + 种子数据 |
| `mysql-p1-upgrade.sql` | approval_node + template_field + 新字段 |
| `mysql-p3-bcrypt.sql` | 密码 BCrypt 加密迁移 |
| `mysql-p4-parallel.sql` | 并行审批（sign_type + approver_ids + approval_task 表） |
| `mysql-p5-timeout.sql` | 超时升级（timeout 相关字段） |

### 2.2 后端启动

```bash
# 进入项目根目录
cd smartoa

# 方式一：Maven Wrapper（推荐，无需安装 Maven）
cd backend && ./mvnw spring-boot:run

# 方式二：Maven
cd backend && mvn spring-boot:run

# 方式三：先构建再运行
cd backend && ./mvnw clean package -DskipTests
java -jar backend/target/smartoa-0.0.1-SNAPSHOT.jar
```

**验证：** 访问 `http://localhost:8080/api/user/current`，返回 401 表示启动正常。

### 2.3 前端启动

```bash
# 进入前端目录
cd smartoa/frontend

# 安装依赖（首次）
pnpm install

# 启动开发服务器
pnpm run dev
```

**验证：** 访问 `http://localhost:5173`，应显示登录页。

## 4. 生产部署

### 4.1 前端构建

```bash
cd frontend
pnpm run build
# 输出到 frontend/dist/
```

### 4.2 后端构建

```bash
cd backend && ./mvnw clean package -DskipTests
# 输出到 target/smartoa-0.0.1-SNAPSHOT.jar
```

### 4.3 部署方式

**方案一：前后端分离（Nginx 反向代理）**

```
nginx.conf:
server {
    listen 80;
    server_name oa.example.com;

    # 前端静态资源
    location / {
        root /var/www/smartoa/dist;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 代理
    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

**方案二：JAR 内嵌静态资源**

将 `frontend/dist/` 内容复制到 `src/main/resources/static/`，重新打包。Spring Boot 会自动托管前端文件。

## 6. 常见问题

### Q1: 启动后端报 "Access denied for user"

**解决：** 检查 `application.properties` 中 MySQL 用户名密码是否正确。默认 root/123456。

### Q2: 前端页面空白

**解决：** 
1. 确保后端已启动且运行在 8080 端口
2. 浏览器 F12 → Console 查看是否有 CORS 错误
3. 检查 Vite 代理是否正常工作

### Q3: 登录后提示"用户不存在"

**解决：** 确认已执行数据库迁移脚本，种子数据已插入。

### Q4: 提交申请提示"尚未分配直属领导"

**解决：** `sys_user` 表中确认该用户的 `direct_leader_id` 已设置。当前种子数据中 zongjian1/zongjian2 无直属领导。

### Q5: 审批后状态不变

**解决：** 确认用正确的审批人账号登录。提交申请后，`current_approver_id` 为直属领导的 ID（SINGLE 模式），或检查 `approval_task` 表（并行模式）。

### Q6: ECharts 图表不显示

**解决：** 确认已执行 `pnpm install`，echarts 依赖已安装。

### Q7: 超时自动处理未触发

**解决：** 
1. 确认 `@EnableScheduling` 已在启动类上配置
2. 检查节点是否正确配置了 `timeoutHours` 和 `timeoutAction`
3. 超时检查每 5 分钟执行一次，需等待对应时间

### Q8: 并行审批任务不可见

**解决：** 确认已执行 `mysql-p4-parallel.sql`，`approval_task` 表已创建。
