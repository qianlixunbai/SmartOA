# 产品需求文档 (PRD)

## SmartOA — 简易 OA 审批流管理系统

---

## 1. 产品概述

### 1.1 产品定位

SmartOA 是一款面向中小企业的轻量级 OA 审批流管理系统，核心解决企业内部请假、报销、加班等日常审批场景的线上化需求。系统以"模板配置 + 多级审批 + 流程引擎"为核心架构，支持条件分支、并行审批（会签/或签）、超时自动升级等高级特性。

### 1.2 产品目标

| 目标 | 说明 |
|---|---|
| 线上化审批 | 替代纸质/邮件/微信等碎片化审批方式 |
| 灵活配置 | 管理员可自定义审批模板、流程节点、签批模式 |
| 全程可追溯 | 每次审批操作留痕，历史可查 |
| 数据可视化 | 审批数据统计图表，辅助管理决策 |
| 智能流转 | 条件分支 + 超时自动升级，减少人工干预 |

### 1.3 目标用户

| 角色 | 描述 |
|---|---|
| 普通员工 (EMPLOYEE) | 提交申请、查看自己的申请记录、撤回申请 |
| 审批人 (MANAGER) | 审批/拒绝/转派待办申请、查看已处理记录 |
| 管理员 (MANAGER) | 管理审批模板、配置审批节点、导出数据 |

## 2. 功能需求

### 2.1 P0 核心功能（已实现）

| 模块 | 功能 | 说明 |
|---|---|---|
| 登录认证 | JWT 登录/登出 | 用户名密码登录，BCrypt 加密，Bearer Token 鉴权 |
| 审批模板 | 模板 CRUD | 创建/编辑/删除/查看审批模板 |
| 申请提交 | 请假申请 | 选择请假类型、日期范围、填写原因并提交 |
| 审批流转 | 2 级硬编码审批 | P0 固定流转：直属领导 → 部门总监 |
| 待办/已办 | 我的审批页面 | Tab 切换待审批/已处理/我的申请 |
| 审批详情 | 查看+操作 | 查看申请详情，通过/驳回操作 |

### 2.2 P1 增强功能（已实现）

| 模块 | 功能 | 说明 |
|---|---|---|
| 技术升级 | JPA → MyBatis-Plus | 持久层框架替换，更灵活的查询能力 |
| 可配置引擎 | 模板自定义审批节点 | 每个模板可配置 N 级审批节点，任意级数 |
| 审批动作 | 同意/拒绝/撤回/转派 | 4 种完整审批操作 |
| 统计报表 | ECharts 可视化 | 平均审批时长柱状图 + 模板使用量饼图 |
| 数据导出 | Excel 导出 | 管理员可导出全部请假单为 .xlsx |
| 前端升级 | 动态步骤条/时间线 | el-steps 动态读取审批节点，Timeline 展示完整流转记录 |

### 2.3 P2 高级功能（已实现）

| 模块 | 功能 | 说明 |
|---|---|---|
| 流程编辑器 | 可视化节点配置 | 拖拽排序、动态添加/删除节点、签批模式选择 |
| 条件分支 | SpEL 条件表达式 | 根据 `leaveType`/`days`/`startDate`/`endDate` 动态触发节点 |
| 并行审批 | 会签 (COUNTER_SIGN) | 节点上所有审批人都同意才进入下一步 |
| 并行审批 | 或签 (OR_SIGN) | 任一审批人同意即进入下一步 |
| 超时升级 | 超时自动处理 | ESCALATE（转派）/ AUTO_APPROVE（自动通过）/ AUTO_REJECT（自动驳回） |
| 滞留修复 | 异常状态修复 | 修复 currentApproverId 为 null 的滞留申请 |

### 2.4 审批人类型

| 类型 | 说明 |
|---|---|
| DIRECT_LEADER | 申请人的直属领导 |
| DEPARTMENT_HEAD | 申请人所属部门的部门总监 |
| SPECIFIC_USER | 指定具体用户（单人/多人） |

### 2.5 签批模式

| 模式 | 推进条件 | 说明 |
|---|---|---|
| SINGLE | 单人审批 | 默认模式，一个审批人决定 |
| COUNTER_SIGN | 全部同意 | 会签，所有人通过才推进 |
| OR_SIGN | 任一同意 | 或签，任一人通过即推进 |

### 2.6 审批流程状态

```
提交申请 → PENDING（审批中）
  ├─ 通过（SINGLE）→ 流转下一节点 → ... → APPROVED（已通过）
  ├─ 通过（COUNTER_SIGN）→ 等待其他人 → ... → APPROVED
  ├─ 通过（OR_SIGN）→ 跳过其他人 → 流转下一节点 → ... → APPROVED
  ├─ 驳回 → REJECTED（已驳回）+ 跳过并行任务
  ├─ 撤回 → WITHDRAWN（已撤回）+ 跳过并行任务
  └─ 超时 → ESCALATE/AUTO_APPROVE/AUTO_REJECT
```

## 5. 非功能需求

| 类别 | 要求 |
|---|---|
| 性能 | 页面响应 < 2s，接口响应 < 500ms |
| 安全 | JWT 鉴权，BCrypt 密码加密，密码不返回前端，CORS 限制 |
| 兼容性 | Chrome/Firefox/Edge 最新版 |
| 可用性 | Element Plus UI 规范，表单校验提示，超时警告栏 |
| 数据 | MySQL 8 存储，utf8mb4 字符集，8 张表 |
| 定时任务 | 超时检查每 5 分钟执行一次 |
