## 调用上下文

- **本命令是通用命令**(全 Phase 都可用 · 跟 git-committer / rules-updater 同向)→ **可接前面对话继续也可退出 `claude` 重启**(贴 v1 提示词 + v1 输出场景适合新会话避免污染上下文 · 但学生在某 Phase 流程中临时演化也可接对话)
- **必须切换模型**:**V4 Pro**(提示词演化诊断需深度推理 + 5 要素对照 + 改进路径分析 · **跟 R-XX/G-21/G-20 评估类命令同向**)· 默认 V4 Flash 推理深度不足导致 v2/v3 改进不显著 → 触发 08b §13 Q26 "不算演化"判定 → 05 验收扣分
- **支持 2 类输入场景**:
  - **主场景**(推荐 · 增量调用 · 对齐 08b §10.4 真实流程):2 次调用 · 真实 v2/v3 输出
  - **备用场景**(一次性生成草稿 · 仅供学生 review 用 · 必须实测验证 v2/v3):1 次调用生成 7 段草稿 · 学生跑 AI 拿 v2/v3 实际输出后**手动**替换草稿中的"预期输出"
- **不做什么**:
  - **不只生成到 v2**(必须含 v3 · 否则违反三段式硬约束)
  - **不编造 v2/v3 实际输出**(主场景 v2 实际输出由学生跑 AI 拿到再贴 · 备用场景标注"预期" · 学生必须实测替换)
  - **不接受同 prompt 跑多次充演化**(对齐 08b §13 Q26 · 命令检测时警告学生重做)
  - **不接受只改 1 个细节(如"请用中文")充演化**(同上)

## 任务

学生在课程中需要积累 **≥ 3 个三段式提示词演化记录**(每个由 **v1 提示词 + v1 AI 输出 + 失败原因 + v2 提示词 + v2 实际输出 + v2 仍不足原因 + v3 提示词 + v3 预期输出 = 8 段**组成 · 对齐 08b §10.4 行 2281-2312 真实示例)· 这是 **05 验收占 6 分的硬性要求**。

本命令任务:基于学生贴的 v1/v2 实际提示词 + 实际 AI 输出,**对照 02 §四 N6 提示词 5 要素**(角色 / 任务 / 输入 / 输出格式 / 约束)**诊断失败原因 + 生成 v2/v3 修正建议(完整草稿)**,帮学生快速完成合规三段式演化记录。

## 失败原因诊断维度(对照 02 §四 N6 提示词 5 要素权威源)

> 📌 **5 要素诊断框架**(02 §四 N6 教学源头 · 每维度对照学生 v1 提示词诊断):

### 1. 角色(Role)缺失

- **典型表现**:v1 没说"你是 SpringBoot 3.5.14 全栈助手"等身份 → AI 输出风格散漫(可能输出 PHP / 旧 Spring 版本 / 无版本约束)
- **修复方法**:在 v2 顶部加角色说明 — "你是 SpringBoot 3.5.14 + Vue 3.5.34 全栈助手 · 对齐 CLAUDE.md §一·一 技术栈基线 2026-05-10"
- **注**:CLAUDE.md §一·一 alwaysApply=true 已自动加载技术栈约束 · v2 角色说明是**强化**,跟 CLAUDE.md §一·一 **不冲突**

### 2. 任务(Task)模糊

- **典型表现**:v1 写 "做一下 X" / "帮我写个登录接口" / "改一下 bug" → AI 不知道做到什么程度
- **修复方法**:v2 改为 "基于 docs/PRD.md §3 P0 第 2 项的 <具体功能名>,生成 SpringBoot Controller + Service + Mapper 三件套"

### 3. 输入(Input)不足

- **典型表现**:v1 没说技术栈 / 已有文件 / 角色清单(应在 CLAUDE.md 起手段 + §一·一 已加载)/ 字段约束 / 错误响应码 → AI 编造字段或漏关键约束
- **修复方法**:v2 加输入约束 — "字段:用户名(@NotBlank @Size(3-20))/ 密码(@NotBlank @Size(min=8) @Pattern 强密码)/ 邮箱(@Email)· 错误响应:用户名重复返回 409 · 业务异常用 BusinessException(code 1xxx-9xxx)抛出"

### 4. 输出格式(Output)无约束

- **典型表现**:v1 没说"用 Result<T> 包装 / 用 LambdaQueryWrapper 查询 / 用 BCryptPasswordEncoder 加密" → AI 自由发挥导致格式各异(30 个学生 30 种实现)
- **修复方法**:v2 加输出格式 — "Controller 返参用 `Result<T>` 静态工厂(对齐 CLAUDE.md §一·三)· Service 用 `LambdaQueryWrapper` · 密码用 `BCryptPasswordEncoder.encode` 加密 · 抛 `BusinessException(409, '用户名已存在')`"

### 5. 约束(Constraints)缺失

- **典型表现**:v1 没有"先列计划再写代码 / 不编造 API / 中文注释 / 不修改我已有的代码"等硬约束 → AI 直接写代码不解释 / 编造不存在的 API / 改了已有代码
- **修复方法**:v2 加硬约束 — "1. 先告诉我打算分几步 + 每步做什么 · 等我确认 OK 再写代码 · 2. 不修改我已有的代码(除非明确要求)· 3. 中文注释 · 关键逻辑加注释 · 4. 不知道的事情说"不知道"不要编造"(对齐 CLAUDE.md §一·四)

> 📌 **诊断要求**:对照 5 要素**逐一**诊断 v1 缺失维度 · **至少 2 条具体诊断**指向核心缺陷 · 不要只列"角色不足"等套话 · 要给出"v1 没说<具体身份> · 应该说<具体身份>"的具体诊断。
