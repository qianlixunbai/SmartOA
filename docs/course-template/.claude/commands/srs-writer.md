## §二 应用修复模式(R-01 issue 处理 · 二级用法)

### 触发场景

`/srs-reviewer` 完成审核后,docs/PRD.md 中已有 `<!-- R-01-issue-编号: 严重度 - 描述 -->` HTML 注释。此时再次调用本命令进入"应用修复"模式。

### 输入

- **必读**:`docs/PRD.md`(reviewer 已插入注释的版本)
- **必读**:`docs/对话记录/Phase1-R01-SRS-review-XXXX.md`(reviewer 报告 · 含每条 issue 的修复建议)
- 用户调用形式:`/srs-writer 应用修复` 或 `/srs-writer 请扫描 PRD.md 中 R-01 注释逐条修复`

### 输出指令

1. 扫描 docs/PRD.md 中所有 `<!-- R-01-issue-... -->` 注释
2. 对每条注释:
   - 修改对应章节内容(基于 reviewer 报告的修复建议)
   - 把注释改写为 `<!-- R-01-issue-编号: 已修复 - 一句话修复说明 -->`
3. **不要重写整个 PRD.md** —— 只 in-place 改动 issue 涉及的章节,其他原文一字不动
4. 输出 diff(显示每个 issue 的改前/改后对比,而不只是"已修复"汇总)

### 输出自检 checklist(应用修复模式)

- [ ] 所有 R-01 注释都已标记"已修复"(没遗漏)
- [ ] 修复内容覆盖 reviewer 报告的 issue 要点
- [ ] 未涉及 issue 的章节原文一字不动(in-place 修复要求)
- [ ] PRD.md 6 大章节结构(§1-§6 · 含 3.1/3.2/3.3 子章节)未破坏
- [ ] 输出 diff 含改前/改后对比(便于学生 review · 也是 05 验收的"改前改后"证据)
